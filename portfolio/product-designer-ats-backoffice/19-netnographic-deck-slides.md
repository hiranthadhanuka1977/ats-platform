## Slide 2 — Product intent

**TalentHub is built for SME / SMB hiring teams**

**On slide (3 columns)**

| Low cost | Good UX | Fast setup |
|----------|---------|------------|
| Core loop only — no CRM bloat | Resume in 1–2 actions | Days/weeks, not 6+ week implementations |
| No $4K–$25K enterprise stack | Honest pipeline state | Sensible defaults, lean admin |
| Predictable scope | No “back to Excel” days | Local-first, focused backoffice |

**Visual:** Three icon cards; target persona: *20–200 employees, 1–3 recruiters*

**Speaker note:** SMEs need credible hiring workflow without paying for modules they’ll never open.

## Slide 10 — Research → design

**How findings shaped TalentHub Applications**

**On slide (two columns, 6 rows max)**

| Market pain | Design response |
|-------------|-----------------|
| Enterprise cost / bloat | Core loop scope — no CRM marketplace |
| Infinite pipeline | **Week-scoped** cohort toolbar |
| Scattered review | **Single packet page** `/applications/[id]` |
| Scan vs funnel | **Table + pipeline** tabs |
| Dishonest status | **10 statuses**, validated transitions, audit |
| Status vs interview mismatch | **Schedule interview** gate |
| Lost context | **`?from=applications`** back path |
| AI without trust | Advisory scores + human-review notice |

**Visual:** Product screenshot annotated with 3 callouts (pipeline week, packet page, status gate)

**Speaker note:** I didn’t copy Greenhouse — I addressed the same frustrations their users still post about, at a scope a 50-person company could adopt.

## Design tips

- **One hero stat per slide** — don’t crowd (43%, 60%, 65%)
- **Slide 2 + 5** establish SME intent early — don’t bury the “why this product exists”
- **Slide 10** is the payoff — screenshot beats another table
- **Slide 11** shows maturity — deferring enterprise scope is a feature for SME positioning
- Keep competitor names to **4 rows** on stage; full table in appendix
