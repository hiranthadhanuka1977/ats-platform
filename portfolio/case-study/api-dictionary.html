Overview

TalentHub splits HTTP concerns across three runtime surfaces:

  • Central API (apps/api, port 4000) — prefix /api/v1 for auth and candidate registration.
  • My Applications (port 3002) — Next.js route handlers for CV upload, parse, apply, and profile import.
  • Backoffice (port 3001) — Next.js BFF for jobs, pipeline, interviews, and admin lookups via staff session cookies.

Most domain logic lives in Next.js route handlers (BFF pattern). Central Hono routes for jobs, applications, and interviews return stub payloads until migrated.

Authentication

Implementation: apps/api/src/modules/auth/index.ts

POST /api/v1/auth/login

Supports audience: "candidate" (default) and audience: "staff". My Applications must use candidate; backoffice must use staff.

Candidate: rejects pending_verification, disabled, and locked accounts. Locks for 15 minutes after 5 failed attempts.

Staff: uses users table; requires is_active = true. Returns role claims in token payload.

Errors: VALIDATION_ERROR (400), INVALID_CREDENTIALS (401), EMAIL_NOT_VERIFIED / ACCOUNT_DISABLED / ACCOUNT_LOCKED (403).

GET /api/v1/auth/me

Requires Authorization: Bearer <access_token>. Returns candidate profile fields or staff name + role. Backoffice falls back to this endpoint when local JWT verification fails during dev.

POST /api/v1/auth/logout

Returns 204 No Content. Not implemented: POST /auth/refresh, OAuth routes.

Backoffice pipeline endpoints

Next.js route handlers — not central /api/v1. Source: apps/backoffice/src/app/api/backoffice/applications/**

PATCH /api/backoffice/applications/{id}/status

Governed status transitions. Requires reason on reject, withdrawalSource on withdraw, offerAccepted: true for offered → hired. Optional expectedUpdatedAt for optimistic concurrency (409 on conflict).

Key errors: INVALID_STATUS_TRANSITION, INTERVIEW_RECORD_REQUIRED, USE_REOPEN_ACTION, TERMINAL_STATE.

POST …/status/undo

Reverts to previous status when latest event matches and undo is allowed.

POST …/reopen

From rejected only. Target must be under_review or shortlisted; mandatory reason.

GET POST …/interviews

One interview row per application (unique constraint). POST creates row and may advance status to interview_scheduled.

GET …/relevance-score?refresh=1

AI relevance scoring. Requires OPENAI_API_KEY; returns 503 without it.

GET …/attachments/cv · …/attachments/cover-letter

Streams files from storage/cvs/ and storage/cover-letters/. Staff session required.

Application status values

Submitted → Under Review → Shortlisted → Interview Scheduled → Interview Completed → Offered → Hired

Terminal: rejected, withdrawn (rejected allows controlled reopen)

Schema alignment

JSON bodies use camelCase (e.g. postedAt) mapping to Prisma snake_case columns. Public job routes return only published rows not past expires_at.

Security notes

  • Store refresh tokens in httpOnly, Secure, SameSite cookies for browser clients.
  • Rate-limit login, register, resend-otp, verify-email, and forgot-password endpoints.
  • Never return password_hash in any response.
  • Staff routes gated by middleware; candidate routes require Bearer JWT.
  • File upload type and size validation on CV endpoints (10 MB max).
