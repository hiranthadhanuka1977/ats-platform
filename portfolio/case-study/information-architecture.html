Platform overview

TalentHub splits hiring into three Next.js apps plus a central auth API. Each app has a distinct audience and navigation model.

Candidate portal Port 3000 · Anonymous

Public job discovery — browse, search, read job details.

My Applications Port 3002 · Signed in

Account, profile, CV library, apply wizard, track status.

Backoffice Port 3001 · Staff

Jobs, pipeline, candidates, interviews, administration.

Central API Port 4000 · Server

Registration, login, OTP, password reset — no UI.

Shell models

Each app uses a different chrome pattern — header, sidebar, and auth gate.

Candidate portal — marketing shell

│  Skip link → #main-content                                  │
├─────────────────────────────────────────────────────────────┤
│  HEADER — Logo (/) · Candidate Login → My Applications      │
├─────────────────────────────────────────────────────────────┤
│  MAIN (#main-content)                                       │
│  · Hero / filters / job cards  OR  job detail sections      │
├─────────────────────────────────────────────────────────────┤
│  FOOTER — Privacy · Terms · Accessibility (placeholders)    │
└─────────────────────────────────────────────────────────────┘

No sidebar. No session in this app. /login, /register, /dashboard are redirect stubs to My Applications.

My Applications — dashboard shell

┌─────────────────────────────────────────────────────────────┐
│  Skip link → #main-content                                  │
├──────────────┬──────────────────────────────────────────────┤
│  SIDEBAR     │  TOP BAR — product label · user menu         │
│  (primary)   ├──────────────────────────────────────────────┤
│              │  PAGE CONTENT (#main-content)                │
└──────────────┴──────────────────────────────────────────────┘

IA principles

1. Separation of audiences

Public browse, authenticated candidate workspace, and staff operations are separate apps with clear entry points.

2. Application as staff unit of work

/applications/[id] is the adjudication anchor; candidate and job are linked hops, not parallel hierarchies.

3. Throughput vs depth

Pipeline and table for volume; detail pages for decisions; candidate profile when person-level history matters.

4. Honest placeholders

Reports, Settings, and footer legal links appear in nav but may surface "coming soon" until shipped.

5. URL-driven discovery state

Job filters use query params so results are shareable and bookmarkable on portal and My Applications.

6. Time-scoped pipeline

Backoffice week navigation reduces noise; table view remains all-time.

7. Administration as reference data

Lookup tables grouped under one nav item with in-page section picker, not eight top-level items.
