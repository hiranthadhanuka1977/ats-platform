Form detail

  • Login: Labels and autocomplete good; global error lacks role="alert"; field errors not linked via aria-describedby.
  • Register: Terms checkbox required; Privacy/Terms referenced as plain text, not links.
  • Apply flow: Many labeled fields; validation summary should use role="alert" and per-field aria-invalid.
  • CV import: Processing overlay accessible; no focus trap while processing.

What is working well

  • Consistent skip navigation and landmark structure across three apps
  • Candidate portal list semantics and live results count
  • Job posting form in backoffice — strong reference for accessible forms
  • Pipeline “Move to” menu — non-drag status path aligned with WCAG 2.2 §2.5.7
  • Schedule interview modal — duration chips, timezone select, live candidate preview
  • AI bias awareness — keyboard-reachable human-review guidance on relevance scores
  • Candidate portal and my-applications use next/font (no third-party font leak)

This audit is the Stage ④ validation artefact in the AI-assisted development process. Re-run after each major UI increment or before portfolio / production sign-off.

Recommended verification

Not done in this audit pass:

  1. axe DevTools or Lighthouse on /, /jobs/{slug}, /login, /register, /dashboard, /applications, /applications/{id}
  2. Keyboard-only walkthrough: pipeline drag alternative, all modals, schedule interview, AI bias flyover
  3. Screen readers: NVDA (Windows), VoiceOver (macOS/iOS) on reject modal and apply form errors
  4. 200% zoom on pipeline board and mobile filter drawer
